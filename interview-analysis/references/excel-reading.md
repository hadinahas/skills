# قراءة ملفات Excel للمقابلات

## كود القراءة الأساسي

```python
from openpyxl import load_workbook

wb = load_workbook('/mnt/user-data/uploads/FILENAME.xlsx', read_only=True)
print('Sheets:', wb.sheetnames)

for sheet in wb.sheetnames:
    ws = wb[sheet]
    print(f'\n--- Sheet: {sheet} ---')
    for i, row in enumerate(ws.iter_rows(values_only=True)):
        if i > 200: break
        if any(v is not None for v in row):
            print(row)
```

## هيكل الملفات المعتاد

تحتوي ملفات المقابلات عادةً على الأوراق التالية:
- **Phone call interview / Phone Interview** — المقابلة الهاتفية الأولية
- **Technical Interview - [اسم المقابل]** — مقابلة أو أكثر تقنية
- **Interview / Interviews** — المقابلات السلوكية الرئيسية (عدة مقابلين جنباً لجنب)

## الحقول المهمة بالترتيب

| الحقل | الأهمية | الإجراء |
|-------|---------|---------|
| `اجابة المرشح` | ★★★★★ | مصدر الحقيقة الوحيد — اقرأ بعناية |
| `رأي المقابل` | ★★★ | مدخل واحد — تحدَّه إذا تعارض مع الخام |
| `التقييم` | ✗ | تجاهل تماماً |
| `الجواب المطلوب` | ★★ | لفهم ما توقعه المقابل فقط |
| `السؤال` | ★★★ | لتقييم جودة السؤال نفسه |

## حالات خاصة

**ورقة تقنية بلا آراء مكتوبة:**
قارن `اجابة المرشح` مع `الجواب المطلوب` بنفسك وقيّم الفجوة.

**درجات بلا إجابات مكتوبة:**
يعني أن المقابل لم يدوّن. لا تستنتج شيئاً من الدرجة وحدها.

**خانات فارغة في إجابة المرشح:**
إذا كانت الخانة فارغة تماماً في سؤال مهم ← سجّله كغياب لا كإجابة صفرية. قد يعني أن السؤال لم يُطرح أو أن المرشح لم يُجب.

## تنبيهات تقنية

- استخدم `read_only=True` دائماً لتجنب تحميل الملف كاملاً في الذاكرة
- لا تثق بـ `ws.max_row` في وضع القراءة — قد تكون قيمة `None` أو خاطئة
- بعض الخلايا تحتوي صيغ (`=AVERAGE(...)`) — تجاهلها تماماً
